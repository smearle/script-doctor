(
  DISCREPANCY: When "cancel" fires during an "again" sequence, what
  state does the level revert to?

  JS behaviour:
    Each "again" iteration is a fresh processInput call.
    bak = backupLevel() captures the state BEFORE this processInput.
    If cancel fires, DoUndo(bak) reverts to the start of THIS
    again-iteration only.  Changes from prior again-iterations persist.

  JAX behaviour:
    In apply_turn, prev_carry holds the state at the start of the
    current while_loop iteration.  If cancelled, we revert to
    prev_carry[0].  This should match JS -- prior iterations' changes
    persist.

  Setup:
  - A Blob slides right via "again".
  - A Trap sits 3 cells to the right of the Blob.
  - When the Blob lands on the Trap, "cancel" fires.

  Expected result:
  - The Blob should end up one cell LEFT of the Trap.
    (The iteration that moved it onto the Trap is cancelled, but
     prior again-iterations that moved it toward the Trap persist.)

  If reversion goes too far back (to before ANY again iterations),
  the Blob stays at its original position -- that would be a bug.
)
