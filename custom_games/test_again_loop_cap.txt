(
  DISCREPANCY: neither JS nor JAX cap the outer "again" loop.
  JS relies on a dry-run check to break the loop when the next turn
  would produce no changes, but if every turn DOES produce changes,
  the loop runs forever.

  In this game two markers on separate layers toggle every turn via
  "again".  Each turn flips A->B and B->A in a two-cell pattern so
  the level is always changing.  Without an iteration cap the game
  hangs in both engines.

  Expected behaviour after fix: the again loop should be capped
  (e.g. 50 iterations) so the game eventually stops.
)
